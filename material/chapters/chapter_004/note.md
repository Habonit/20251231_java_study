## 노트 작성

> 수업 내용 정리. 자유롭게 작성.

- 객체 지향성을 갖추기 위해선 this method를 써서 데이터는 하나의 클래스에서 다루는 게 좋다. 

- this를 생략하는 것도 scope의 관계에 따라 가능할 때도 있습니다. 그러나 이를 반드시 적는 코딩 스타일도 있습니다. 

- 생성자 이름과 클래스 이름을 일치시키고 this로 받아주면 한 번에 데이터 생성 가능.

- 또, 배열로 생성할 때 클래스[] 배열이름 = {요소1, 요소N} 이렇게 넣어도 가능

- 생성자가 있으면 유령 회원 문제를 방지하고 컴파일 단계에서 에러를 내기 때문에 유령회원 문제를 개선할 수 있음. 

- 생성자가 없으면 자바가 직접 기본 생성자를 만들어준다. 생성자는 반드시 한 개라도 있어야 합니다. 생성자가 있으면 자바는 기본 생성자를 만들지 않습니다.

- this()는 생성자 코드 안에서 첫 줄에서만 실행 가능. 

- this()는 생성자 내부에서 같은 클래스의 다른 생성자를 호출할 때 사용하는 키워드입니다. 시그니처란 메서드나 생성자를 구별하는 고유한 식별 정보로, 이름과 매개변수의 타입, 개수, 순서로 구성됩니다. 따라서 this()를 호출하면 컴파일러는 전달된 인자의 시그니처와 일치하는 생성자를 찾아 호출하며, 코드의 작성 순서는 영향을 주지 않습니다.
