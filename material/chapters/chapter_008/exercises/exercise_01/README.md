# 연습 문제 01: final 참조형 변수와 객체 내부 값 변경

## 학습 목표

- `final` 참조형 변수의 특성을 이해한다
- 참조값(주소) 변경 불가와 참조 대상의 필드 변경 가능의 차이를 구분한다
- `final` 키워드의 실제 동작 방식을 코드로 확인한다

---

## 문제 설명

`Person` 클래스는 `final Address address` 필드를 가지고 있다.
이 상황에서 다음을 확인해보자:

1. `final` 참조형 변수에 **새로운 객체를 할당**할 수 있는가?
2. `final` 참조형 변수가 가리키는 **객체의 내부 필드**를 변경할 수 있는가?

## 핵심 포인트

```
final Address address = new Address("서울시", "강남구");

// 1. 참조값 변경 -> 불가능 (컴파일 오류)
address = new Address("부산시", "해운대구");  // X

// 2. 참조 대상의 필드 변경 -> 가능
address.city = "부산시";      // O
address.district = "해운대구"; // O
```

**기억하세요**: `final`은 "변수가 담고 있는 값"만 변경 못하게 막는다.
참조형 변수의 경우, 그 값은 **객체의 주소(참조값)**이다.
따라서 **다른 객체를 가리키게 하는 것**만 막을 뿐, **기존 객체 내부의 변경**은 막지 않는다.
