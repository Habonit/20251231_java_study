# 연습 문제 3: 변수 스코프와 생명주기

## 난이도: 심화

## 학습 목표

- 멤버 변수와 지역 변수의 차이를 이해한다
- 변수의 스코프(유효 범위)를 이해한다
- GC(Garbage Collection)의 기본 개념을 이해한다

---

## 문제 설명

멤버 변수와 지역 변수가 어떻게 다르게 동작하는지,
그리고 변수의 스코프가 어떻게 결정되는지 확인하는 프로그램을 작성하라.

---

## 핵심 개념

### 멤버 변수 vs 지역 변수

| 구분 | 멤버 변수 | 지역 변수 |
|:-----|:-----|:-----|
| 선언 위치 | 클래스 내부 (메서드 외부) | 메서드 내부 |
| 생명 주기 | 객체가 존재하는 동안 | 메서드 실행 중에만 |
| 초기화 | 자동 초기화 (int=0, String=null) | 수동 초기화 필수 |
| 스코프 | 객체 전체에서 접근 가능 | 선언된 블록 {} 내에서만 |

### 스코프(Scope)란?

- 변수가 유효한 범위
- Java에서는 `{}` (중괄호) 블록 단위로 스코프가 정해진다
- 블록을 벗어나면 변수에 접근할 수 없다

### GC(Garbage Collection)

- 아무도 참조하지 않는 객체를 메모리에서 자동 제거
- 참조가 끊긴 객체는 GC의 대상이 됨
- 개발자가 직접 메모리를 해제할 필요 없음 (Java의 장점)

---

## 요구 사항

1. `Counter.java`에 `Counter` 클래스를 정의하라
   - count (int): 멤버 변수 - 카운터 값

2. `Main.java`에서:
   - 멤버 변수와 지역 변수의 차이 실험
   - 블록 스코프 실험
   - 참조가 끊긴 객체가 GC 대상이 됨을 설명

---

## 예상 출력

```
=== 멤버 변수 vs 지역 변수 ===
Counter 객체 생성 (멤버 변수 count는 자동으로 0으로 초기화)
counter.count = 0
counter.count를 5로 변경
counter.count = 5
increment() 메서드 호출 (내부 지역 변수 사용)
counter.count = 6

=== 블록 스코프 실험 ===
if 블록 내부: blockVar = 100
if 블록 외부: blockVar는 접근 불가 (컴파일 에러)

=== GC 대상 실험 ===
data1 객체 생성
data1 = null (참조 끊음)
-> data1이 가리키던 객체는 이제 GC 대상
```

---

## 힌트

- 멤버 변수: 클래스 내부, 메서드 외부에 선언
- 지역 변수: 메서드 내부에 선언, 반드시 초기화 필요
- 스코프: `{}` 블록 내에서만 유효
- GC: `객체 = null;`로 참조를 끊으면 GC 대상이 됨
