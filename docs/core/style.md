# 문서 작성 스타일 가이드

이 문서는 프로젝트 내 마크다운 문서의 작성 스타일 표준을 정의한다.

---

## 1. 문서 구조

### 1.1 제목 레벨

| 레벨 | 용도 | 예시 |
|:-----|:-----|:-----|
| `#` | 문서 제목 (파일당 1개) | `# Project Constitution` |
| `##` | 주요 섹션 | `## 1. Git Branch Convention` |
| `###` | 하위 섹션 | `### 1.1 핵심 철학` |
| `####` | 세부 항목 | `#### 1.1.1 상세 내용` |

### 1.2 섹션 번호링

- 주요 섹션(`##`)은 **숫자 번호**를 포함한다
- 하위 섹션(`###`)은 **점 표기법**을 사용한다
- 번호와 제목 사이에 **마침표와 공백**을 넣는다

```markdown
## 1. 섹션 제목
### 1.1 하위 섹션
### 1.2 하위 섹션
## 2. 다음 섹션
### 2.1 하위 섹션
```

### 1.3 구분선

- 주요 섹션(`##`) 사이에 `---` 구분선을 사용한다
- 문서 시작 부분에는 구분선을 넣지 않는다

```markdown
## 1. 첫 번째 섹션

내용...

---

## 2. 두 번째 섹션

내용...
```

---

## 2. 테이블

### 2.1 기본 형식

- 헤더와 구분자를 포함한다
- 좌측 정렬(`:-----`)을 기본으로 사용한다
- 구분자의 하이픈은 **5개**를 사용한다

```markdown
| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| 내용1 | 설명1 | 예시1 |
| 내용2 | 설명2 | 예시2 |
```

### 2.2 정렬 규칙

| 정렬 | 구문 | 용도 |
|:-----|:-----|:-----|
| 좌측 정렬 | `:-----` | 텍스트 (기본값) |
| 중앙 정렬 | `:----:` | 상태, 아이콘 |
| 우측 정렬 | `-----:` | 숫자, 버전 |

## 3. 코드 블록

### 3.1 인라인 코드

- 파일명, 함수명, 변수명, 명령어에 백틱(`` ` ``)을 사용한다

```markdown
`constitution.md` 파일을 참조한다.
`git commit` 명령어를 실행한다.
```

### 3.2 코드 블록

- 언어를 **반드시 명시**한다
- 지원 언어: `python`, `java`, `typescript`, `bash`, `json`, `yaml`, `markdown`

````markdown
```python
def hello():
    print("Hello, World!")
```
````

### 3.3 다이어그램

- ASCII 아트 다이어그램은 언어 없이 코드 블록으로 감싼다

````markdown
```
┌─────────┐     ┌─────────┐
│  시작   │ ──► │  종료   │
└─────────┘     └─────────┘
```
````

## 4. 리스트

### 4.1 비순서 리스트

- 하이픈(`-`)을 사용한다
- 들여쓰기는 **스페이스 2칸**

```markdown
- 항목 1
- 항목 2
  - 하위 항목 2.1
  - 하위 항목 2.2
- 항목 3
```

### 4.2 순서 리스트

- 숫자와 마침표를 사용한다

```markdown
1. 첫 번째 단계
2. 두 번째 단계
3. 세 번째 단계
```

### 4.3 체크리스트

- `- [ ]` (미완료), `- [x]` (완료)를 사용한다

```markdown
- [ ] 미완료 항목
- [x] 완료 항목
```

## 5. 텍스트 강조

### 5.1 강조 규칙

| 스타일 | 구문 | 용도 |
|:-----|:-----|:-----|
| **굵게** | `**텍스트**` | 중요 용어, 강조 |
| *기울임* | `*텍스트*` | 부가 설명 |
| `코드` | `` `텍스트` `` | 코드, 명령어, 파일명 |

### 5.2 인용문

- `>` 기호를 사용하여 인용 또는 노트를 표현한다

```markdown
> 이것은 인용문 또는 중요한 노트이다.
```

## 6. 언어 규칙

### 6.1 기본 언어

- 모든 문서는 **한글**을 기본으로 작성한다
- 문장은 **경어체**가 아닌 **평어체**로 작성한다 (예: "~한다", "~이다")

### 6.2 영어 사용

- **기술 용어**는 영어 원문을 유지한다
  - 예: DataFrame, API, cache, Mock, Stub
- **명령어**는 영어 원문을 유지한다
  - 예: `git commit`, `npm install`
- **불필요한 영어 사용을 지양**한다

```markdown
# 좋은 예
데이터를 DataFrame으로 변환한다.

# 나쁜 예
Convert data to DataFrame.  ← 불필요한 영어
```

## 7. 링크 및 참조

### 7.1 내부 링크

- 상대 경로를 사용한다

```markdown
[프로젝트 규칙](./constitution.md)
[API 명세](../api_spec.md)
```

### 7.2 외부 링크

- 전체 URL을 사용한다

```markdown

```

### 7.3 섹션 링크

- 앵커 링크를 사용한다

```markdown
[테이블 섹션](#2-테이블)
```

## 8. 특수 요소

### 8.1 상태 표시

| 아이콘 | 의미 |
|:-----|:-----|
| ✅ | 완료 |
| 🚧 | 진행중 |
| 📋 | 예정 |
| ⚠️ | 주의/경고 |
| ❌ | 실패/차단 |

### 8.2 우선순위 표시

| 아이콘 | 의미 |
|:-----|:-----|
| 🔴 | P0 - 필수/긴급 |
| 🟡 | P1 - 중요 |
| 🟢 | P2 - 개선 |

### 8.3 변경 유형 표시

| 아이콘 | 의미 |
|:-----|:-----|
| 🔄 | 변경 |
| 🔧 | 개선 |
| ➕ | 추가 |
| ➖ | 제거 |
| 🐛 | 버그 수정 |

## 9. 파일 네이밍

### 9.1 규칙

- **소문자**와 **언더스코어**를 사용한다
- 확장자는 `.md`를 사용한다

```
constitution.md
tdd_constitution_java.md
api_spec_template.md
```

### 9.2 템플릿 파일

- 템플릿 파일은 `_template` 접미사를 붙인다

```
improvement_proposal_template.md
error_spec_template.md
```

## 10. 체크리스트

문서 작성 완료 시 다음 항목을 확인한다:

- [ ] 문서 제목(`#`)이 파일당 1개인가?
- [ ] 섹션 번호가 순차적으로 부여되어 있는가?
- [ ] 주요 섹션 사이에 구분선(`---`)이 있는가?
- [ ] 테이블 정렬이 `:-----` 형식인가?
- [ ] 코드 블록에 언어가 명시되어 있는가?
- [ ] 리스트에 `-` 또는 숫자를 사용했는가?
- [ ] 한글을 기본으로 작성했는가?
- [ ] 기술 용어만 영어를 사용했는가?
